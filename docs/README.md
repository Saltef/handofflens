# Documentation Map

The public documentation surface is intentionally small. Start with the current evidence, then use the archive only when you need the development history behind a design choice.

## Canonical Documents

- [Public Benchmark Run Results](public-benchmark-results-2026-07-21.md) - ACI-Bench note-generation, attribution repair, BioScope assertion runs, and explicit non-claim boundaries.
- [Scientific Write-up](SCIENTIFIC_WRITEUP.md) - problem framing, architecture, findings, and limitations.
- [Claims Register](claims-register.md) - what the evidence allows, what it does not allow, and what remains blocked.
- [Reproducibility](REPRODUCIBILITY.md) - commands, Docker paths, private-input boundaries, and artifact handling.
- [Benchmark Adapter and Scoring](benchmark-adapter-scoring.md) - ACI adapter, note-generation scoring, source-support proxies, and BioScope runners.
- [Records Adapter Contract](records-adapter-contract.md) - stable input schema and publishing rules for dataset adapters.
- [Model Card](../MODEL_CARD.md) - intended use, non-use, and limitations.
- [Data Exposure Attestation](data-exposure-attestation.md) - what is excluded from the public repository.
- [Security Checklist](security-checklist.md) - secret and private-data handling checks.

## Archive

- [Archive Index](archive/README.md) - historical validation snapshots, superseded pipeline reports, exploratory frameworks, conformal-routing work, and process notes.

Archived documents are retained for auditability. They do not override the current README, benchmark results, model card, or claims register. In particular, conformal and selective-routing material remains proxy-label research, not clinical validation.
